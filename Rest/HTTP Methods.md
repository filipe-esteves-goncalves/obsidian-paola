| **Method**  | **Description**                                                                                                                    | **Typical Use Case**                                                                                                                   | Status                                    |
| ----------- | ---------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------- |
| **GET**     | Retrieves data or information from a specified resource. <br>It should only fetch data and have no other effect.                   | Loading a web page or reading a specific item from a database.                                                                         | Recommended                               |
| **QUERY**   | A Get with request body                                                                                                            | **GraphQL APIs:** Sending a multi-kilobyte GraphQL query to a server without having to tunnel it through an uncacheable `POST` request | Recommended                               |
| **POST**    | Submits data to the specified resource to be processed. <br>This often results in a change in state or side effects on the server. | Submitting a contact form or creating a new user account.                                                                              | Recommended                               |
| **PUT**     | Replaces all current representations of the target resource with the uploaded content.                                             | Updating an entire user profile with new information.                                                                                  | Recommended                               |
| **PATCH**   | Applies partial modifications to an existing resource.                                                                             | Updating just the email address of a user profile without sending the rest of the data.                                                | Recommended                               |
| **DELETE**  | Removes the specified target resource.                                                                                             | Deleting a specific comment or closing an account.                                                                                     | Recommended                               |
| **HEAD**    | Requests a response identical to a `GET` request, but without the response body (transfers only the headers).                      | Checking if a file exists, its size, or its last modified date before deciding to download it.                                         | Recommended                               |
| **OPTIONS** | Describes the communication options and allowed HTTP methods for the target resource.                                              | Used by browsers for CORS (Cross-Origin Resource Sharing) preflight requests to see if an API allows access.                           | Required for Modern Web Apps              |
| **CONNECT** | Establishes a network tunnel to the server identified by the target resource.                                                      | Setting up an SSL (HTTPS) connection through an unencrypted HTTP proxy.                                                                | Handled by Proxies / <br>Do Not Implement |
| **TRACE**   | Performs a message loop-back test along the path to the target resource.                                                           | Debugging and network diagnostic purposes to see what intermediate servers are doing to the request.                                   | DISABLE IMMEDIATELY                       |
